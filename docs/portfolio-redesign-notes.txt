PEIYU LIU PERSONAL WEBSITE
FILE ORGANIZATION AND PORTFOLIO REDESIGN NOTES
Date: July 12, 2026


1. ORIGINAL OBJECTIVES
======================

The website is a Hugo personal website using the Anatole theme. The original
objectives were:

1. Make the repository easier to understand and maintain without changing the
   current visible website content.
2. Make it easy to identify which file controls each main navigation tab and
   each content area.
3. Improve the Portfolio page while preserving the existing site's visual
   theme.
4. Place Coding Projects first, followed by a Publications list.
5. Let each coding project open a detailed page containing its workflow,
   explanation, tutorial, and code.
6. Give each publication direct PDF and article-page links.

5. PORTFOLIO DESIGN REQUIREMENTS
================================

Required page order:

1. Portfolio heading
2. Coding Projects section
3. Publications section

Coding Projects:
- Appear at the top of the Portfolio page.
- Use the current Anatole theme's established visual language.
- Each project should include a project image or simple workflow preview,
  project name, status/year, short description, link, and technical tags.
- Clicking a project should open its own detail page.

Project detail pages should support:
- Overview
- Problem or motivation
- Workflow or architecture
- Setup and usage instructions
- Tutorial or worked example
- Code examples
- GitHub repository link
- Demo, documentation, or package website link when available

Publications:
- Appear after Coding Projects on the same Portfolio page.
- Use a compact academic list rather than oversized magazine-cover cards.
- Each row should show year, title, author/citation information, and links.
- Preferred actions are PDF and Article.
- Code or Supplement may be added only when those resources exist.


6. VISUAL DIRECTION AND USER FEEDBACK
=====================================

The first proposed sketch introduced a new product-style visual system with
modern cards, badges, and workflow panels. This direction was rejected because
it added too much visual invention and departed from the existing website.

Important user feedback:
- Do not redesign the website into a different product aesthetic.
- Preserve and refine the existing Anatole theme.
- Improve or combine existing components instead of introducing an unrelated
  design system.
- The result should be compact, elegant, and professional.

Current preferred visual direction:
- Keep the existing left sidebar.
- Keep the existing uppercase text navigation.
- Keep the white background, thin rules, restrained gray surfaces, current
  typography, and subtle blue links.
- Reduce excessive empty space and oversized project images.
- Retain the existing image-plus-description composition, but make each project
  substantially shorter and more balanced.
- Use compact publication rows after the projects.
- Avoid unnecessary badges, dashboards, statistics, decorative panels, and
  colorful product-style UI elements.

A second discussion sketch was created to illustrate this direction. It was a
conversation-only visual reference and was not applied to the website source.


7. CONTACT NAVIGATION DECISION
==============================

Recommendation:
- Remove Contact from the visible top-level navigation.
- Keep the existing Contact page and URL for backward compatibility.
- Add a concise contact call-to-action to the left sidebar.

Hiring-manager and recruiter rationale:
- Recruiters should be able to find and copy the email address from every page.
- A separate Contact tab adds an extra click without adding meaningful content.
- The current Contact page contains too little information to justify a primary
  navigation position.
- Professional work, projects, publications, and tutorials deserve greater
  navigation priority.
- A visible email address is easier to copy into recruiting systems or forward
  to another manager than a generic "Contact me" button.

Recommended sidebar copy:

Open to collaboration
Research, biostatistics, and statistical computing opportunities.
peiyu.liu.stats@gmail.com

Recommended link behavior:

<a href="mailto:peiyu.liu.stats@gmail.com">
  peiyu.liu.stats@gmail.com
</a>

Important implementation details:
- The full email address should remain visible.
- On mobile, the contact block must remain easy to find when the sidebar
  collapses or moves.
- The old /contact/ URL should not be deleted even if the navigation item is
  hidden.

Recommended final top navigation:
- ABOUT
- POSTS
- PORTFOLIO
- TUTORIALS
